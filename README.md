# VideoBot

بوت Discord لعرض الفيديوهات داخل Embed مع أزرار مشاهدة ونسخ الرابط.

## البحث التلقائي في يوتيوب

`/addvideo` ما يحتاج رابط إجباري — لو تركت `video_url` فاضي، البوت يبحث تلقائياً في يوتيوب باسم العنوان اللي كتبته ويجيب أول نتيجة (العنوان، الوصف، الصورة، المدة، ورابط نظيف بدون باراميترات زايدة).

يستخدم مكتبة `yt-search` (بدون API key أو فيزا). ملاحظة: بما إنها تسحب البيانات من صفحة يوتيوب مباشرة، بعض بيئات الاستضافة (زي Render) ممكن يوتيوب يقيّد الطلبات القادمة من IP السيرفر أحياناً. لو صار كذا، البوت يرد برسالة واضحة تطلب منك تحط الرابط يدوياً بدل ما يفشل بصمت.

## عرض الفيديو مشغّل داخل ديسكورد

أمر `/video` يرسل رسالتين:
1. **Embed** فيه العنوان، الوصف، المدة، التصنيف، والصورة المصغرة + أزرار (مشاهدة / نسخ الرابط)
2. **رسالة ثانية منفصلة** فيها رابط الفيديو الخام فقط — هذي الرسالة هي اللي تخلي ديسكورد يشغّل مشغل الفيديو المدمج (سواء معاينة يوتيوب أو مشغل mp4) تحتها مباشرة، لأن الـ Embed العادي ما يقدر "يحمل" فيديو قابل للتشغيل داخله.

**لماذا الرابط لازم يكون نظيف؟** روابط يوتيوب فيها باراميترات زيادة (`&t=`, `&list=`, `&si=`) أحياناً تكسر المعاينة أو تفتح بلاي لست بدل الفيديو. لهذا كل رابط يتولّد من البحث التلقائي يكون بصيغة نظيفة: `youtube.com/watch?v=VIDEO_ID` بس.

## التشغيل على Termux

```bash
cd VideoBot
npm install
```

1. افتح ملف `.env` وحط بياناتك:
   - `DISCORD_TOKEN` → من [Discord Developer Portal](https://discord.com/developers/applications) > Bot > Token
   - `CLIENT_ID` → Application ID من نفس الصفحة
   - `GUILD_ID` → (اختياري) آيدي السيرفر اللي تسوي فيه تجارب، عشان الأوامر تظهر فوراً بدل ما تنتظر ساعة

2. شغّل البوت:
```bash
npm run start
```
البوت يتحقق تلقائياً من الأوامر المسجلة عند ديسكورد عند كل تشغيل: إذا كانت مطابقة (نفس العدد والمحتوى) ما يعيد التسجيل، وإذا فيه فرق (أمر جديد، محذوف، أو معدّل) يسجّلها تلقائياً بدون أي خطوة يدوية.

يمديك أيضاً تسجّل الأوامر يدوياً بشكل منفصل وقت ما تبي (نفس المنطق، بدون تشغيل البوت):
```bash
npm run deploy
```

## الأوامر المتوفرة

| الأمر | الوصف |
|---|---|
| `/addvideo` | إضافة فيديو جديد |
| `/editvideo` | تعديل فيديو موجود |
| `/deletevideo` | حذف فيديو |
| `/listvideos` | عرض كل الفيديوهات |
| `/video` | عرض فيديو معيّن مع Embed وأزرار |

## النشر على Render

1. ارفع المشروع على GitHub (تأكد إن `.env` و `database.db` داخل `.gitignore` — موجودين أصلاً).
2. في Render، أنشئ **Web Service** جديد واربطه بالـ repo.
3. Build Command: `npm install`
4. Start Command: `node index.js`
5. في تبويب **Environment**، أضف نفس المتغيرات اللي في `.env` (DISCORD_TOKEN, CLIENT_ID, GUILD_ID).

### ⚠️ ملاحظة مهمة عن قاعدة البيانات
`database.db` يتخزن محلياً على السيرفر. Render (خصوصاً الخطة المجانية) يستخدم **نظام ملفات مؤقت** — يعني كل إعادة تشغيل أو نشر جديد (deploy) بيمسح البيانات المخزنة. هذا قرار واعي منك حسب ما اتفقنا عليه، فقط تأكد إنك متوقع هالسلوك ومرتاح له.

### سيرفر النبض (Health Check)
البوت فيه سيرفر HTTP بسيط (`health.js`) يفتح بورت ويرجع رد JSON على `/`:
```json
{ "status": "ok", "bot": "VideoBot", "time": "..." }
```
هذا يخليك تقدر تسوي الخدمة على Render نوعها **Web Service** عادي (بدل Background Worker)، ويرجع لك Render رابط تقدر تستخدمه مع خدمة Ping خارجية (زي UptimeRobot أو Cron-job.org) عشان تبعت طلب كل فترة وتمنع الخدمة من "النوم" في الخطة المجانية.

- البورت يُقرأ تلقائياً من متغير البيئة `PORT` اللي يوفره Render، أو `3000` محلياً كافتراضي.
- ما تحتاج تضيف `PORT` يدوياً في Environment Variables — Render يحطه تلقائي.
